Experiment No. 2 on Some semantic elements used in HTML 5 and their functions.

Web Programming

With HTML4, developers used their own id/class names to style elements: header, top, bottom, footer, menu, navigation, main, container, content, article, sidebar, topnav, etc. This made it impossible for search engines to identify the correct web page content. With the new HTML5 elements (<header> <footer> <nav> <section> <article>), this will become easier. A Semantic Web Allows data to be shared and reused across applications, enterprises, and communities.
nav (short for navigation) element -The <nav>element defines a set of navigation links.
A <section> is a thematic grouping of content, typically with a heading.
ARTICLE TAG
The <article> element specifies independent, self-contained content.
An article should make sense on its own, and it should be possible to read it independently from the rest of the web site.
The <div> tag defines a division or a section in an HTML document. The <div> tag is used to group block-elements to format them with CSS.
The <aside> element defines some content aside from the content it is placed in (like a sidebar). The aside content should be related to the surrounding content.
